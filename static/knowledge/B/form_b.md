# 帳票B作成：審査・判定

## 概要

帳票Bは、帳票Aの成果物をレビューし、プロジェクトの進行可否を判定する重要なゲートタスクです。
このドキュメントは、帳票B作成の実施方法、判定基準、および過去事例を記載しています。

## 実施内容

### 1. 帳票Aのレビュー

- 基本情報の完全性確認
- 顧客要件の妥当性確認
- リスク評価の適切性確認
- スケジュール案の現実性確認

### 2. 審査判定

- QMS基準への適合性確認
- 関連部門のレビュー意見の集約
- 改善指摘事項の整理
- Go/No-Go判定

## 判定基準

### Go条件（進行可）

- [ ] 基本情報が100%完全
- [ ] 顧客要件の確認が完了
- [ ] 主要リスクが識別されている
- [ ] 開発スケジュールが妥当

### No-Go条件（進行不可）

- [ ] 必須情報が不足している
- [ ] 顧客要件が曖昧
- [ ] 対応不可能なリスクが存在

## 承認フロー

1. レビュー実施（制御開発チーム）
2. 品質部門確認
3. プロジェクトマネージャー承認
4. 経営層承認（必要に応じて）

## 過去事例

### No-Go判定事例：プロジェクトY

プロジェクト初期段階でNo-Go判定が下りたことで、
後々の大きなトラブルを未然に防ぐことができた。
